## Step 1: Authentication

First, you need to authenticate to get an access token. Use the login endpoint:

<CodeGroup>

```bash cURL
curl -X POST https://api.railsfromthecrypt.com/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "your-email@example.com",
    "password": "your-password"
  }'
```

```javascript JavaScript
const response = await fetch('https://api.railsfromthecrypt.com/v1/auth/login', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    email: 'your-email@example.com',
    password: 'your-password'
  })
});

const data = await response.json();
console.log(data);
```

```python Python
import requests

response = requests.post(
    'https://api.railsfromthecrypt.com/v1/auth/login',
    json={
        'email': 'your-email@example.com',
        'password': 'your-password'
    }
)

data = response.json()
print(data)
```

</CodeGroup>

<ResponseExample>

```json Response
{
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "refresh_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "expires_in": 7200,
  "is_2fa_enabled": false
}
```

</ResponseExample>

<Warning>
Save your access token securely. It expires after 2 hours, and you'll need to use the refresh token to get a new one.
</Warning>

## Common Issues

<AccordionGroup>
  <Accordion title="Authentication Errors">
    **401 Unauthorized**: Check that your access token is valid and not expired. Use the refresh token endpoint to get a new access token.
    
    **403 Forbidden**: Your user account may not have sufficient permissions for the requested operation.
  </Accordion>
  <Accordion title="Rate Limiting">
    **429 Too Many Requests**: You've exceeded the rate limit. Wait for the reset time indicated in the response headers before making more requests.
  </Accordion>
  <Accordion title="Network Issues">
    **Connection Timeout**: Check your internet connection and ensure you're using the correct base URL for your environment.
  </Accordion>
</AccordionGroup>

## Environment Variables

For production applications, use environment variables to store your credentials:

<CodeGroup>

```bash .env
HR_API_BASE_URL=https://api.hashrails.xyz
HR_API_EMAIL=your-email@example.com
HR_API_PASSWORD=your-password
HR_ACCESS_TOKEN=your-access-token
```

```javascript JavaScript
const config = {
  baseUrl: process.env.HR_API_BASE_URL,
  email: process.env.HR_API_EMAIL,
  password: process.env.HR_API_PASSWORD,
  accessToken: process.env.HR_ACCESS_TOKEN
};
```

```python Python
import os

config = {
    'base_url': os.getenv('HR_API_BASE_URL'),
    'email': os.getenv('HR_API_EMAIL'),
    'password': os.getenv('HR_API_PASSWORD'),
    'access_token': os.getenv('HR_ACCESS_TOKEN')
}
```

</CodeGroup>

<Warning>
Never commit API credentials to version control. Use environment variables or secure credential management systems.
</Warning>
