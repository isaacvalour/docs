## Overview

Refresh an expired access token using a valid refresh token. This endpoint allows users to get a new access token without re-authenticating.

<Note>
  Refresh tokens have a longer lifespan than access tokens and should be stored securely.
</Note>

## Request

<ParamField body="refresh_token" type="string" required>
  Valid refresh token obtained during login.
</ParamField>

## Response

<ResponseField name="success" type="boolean">
  Indicates if the token refresh was successful.
</ResponseField>

<ResponseField name="message" type="string">
  Success message.
</ResponseField>

<ResponseField name="data" type="object">
  Token data object.
</ResponseField>

<ResponseField name="data.access_token" type="string">
  New access token.
</ResponseField>

<ResponseField name="data.refresh_token" type="string">
  New refresh token (optional, may be the same as the old one).
</ResponseField>

<ResponseField name="data.expires_in" type="number">
  Access token expiration time in seconds.
</ResponseField>

<ResponseField name="data.token_type" type="string">
  Token type, typically "Bearer".
</ResponseField>

## Example Request

<CodeGroup>

```bash cURL
curl -X POST "https://api.railsfromthecrypt.com/v1/auth/refresh" \
  -H "Content-Type: application/json" \
  -d '{
    "refresh_token": "your_refresh_token_here"
  }'
```

```javascript JavaScript
const response = await fetch('https://api.railsfromthecrypt.com/v1/auth/refresh', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    refresh_token: 'your_refresh_token_here'
  })
});

const data = await response.json();
console.log(data);
```

```python Python
import requests

response = requests.post(
    'https://api.railsfromthecrypt.com/v1/auth/refresh',
    json={
        'refresh_token': 'your_refresh_token_here'
    }
)

data = response.json()
print(data)
```

## Notes

- Refresh tokens typically have a longer lifespan than access tokens (e.g., 30 days vs 1 hour)
- The old refresh token may be invalidated when a new one is issued (token rotation)
- Store refresh tokens securely and never expose them in client-side code
- Implement automatic token refresh in your application to maintain seamless user experience

## Implementation Example

<CodeGroup>

```javascript JavaScript
class TokenManager {
  constructor() {
    this.accessToken = localStorage.getItem('access_token');
    this.refreshToken = localStorage.getItem('refresh_token');
  }
  
  async refreshAccessToken() {
    try {
      const response = await fetch('https://api.railsfromthecrypt.com/v1/auth/refresh', {
        method: 'POST',
        headers: {
          'Content-Type': 'application/json'
        },
        body: JSON.stringify({
          refresh_token: this.refreshToken
        })
      });
      
      if (!response.ok) {
        throw new Error('Failed to refresh token');
      }
      
      const data = await response.json();
      
      // Update stored tokens
      this.accessToken = data.data.access_token;
      this.refreshToken = data.data.refresh_token;
      
      localStorage.setItem('access_token', this.accessToken);
      localStorage.setItem('refresh_token', this.refreshToken);
      
      return this.accessToken;
    } catch (error) {
      // Refresh failed, redirect to login
      this.clearTokens();
      window.location.href = '/login';
      throw error;
    }
  }
  
  clearTokens() {
    this.accessToken = null;
    this.refreshToken = null;
    localStorage.removeItem('access_token');
    localStorage.removeItem('refresh_token');
  }
}
```

</CodeGroup>
