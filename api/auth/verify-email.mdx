## Overview

Verify a user's email address using the verification token sent to their email. This endpoint is called when a user clicks the verification link in their email.

<Note>
  Email verification is required before users can access most API endpoints.
</Note>

## Request

<ParamField body="token" type="string" required>
  Email verification token received via email.
</ParamField>

<ParamField body="email" type="string" required>
  Email address to verify.
</ParamField>

## Response

<ResponseField name="success" type="boolean">
  Indicates if the email verification was successful.
</ResponseField>

<ResponseField name="message" type="string">
  Success message.
</ResponseField>

<ResponseField name="data" type="object">
  User data object.
</ResponseField>

<ResponseField name="data.id" type="string">
  User identifier.
</ResponseField>

<ResponseField name="data.email" type="string">
  Verified email address.
</ResponseField>

<ResponseField name="data.email_verified" type="boolean">
  Email verification status (should be true).
</ResponseField>

<ResponseField name="data.verified_at" type="string">
  Email verification timestamp.
</ResponseField>

## Example Request

<CodeGroup>

```bash cURL
curl -X POST "https://api.railsfromthecrypt.com/v1/auth/verify-email" \
  -H "Content-Type: application/json" \
  -d '{
    "token": "verification_token_from_email",
    "email": "john.doe@example.com"
  }'
```

```javascript JavaScript
const response = await fetch('https://api.railsfromthecrypt.com/v1/auth/verify-email', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    token: 'verification_token_from_email',
    email: 'john.doe@example.com'
  })
});

const data = await response.json();
console.log(data);
```

```python Python
import requests

response = requests.post(
    'https://api.railsfromthecrypt.com/v1/auth/verify-email',
    json={
        'token': 'verification_token_from_email',
        'email': 'john.doe@example.com'
    }
)

data = response.json()
print(data)
```

</CodeGroup>

## Status Codes

<ResponseField name="200" type="number">
  Email verified successfully
</ResponseField>

<ResponseField name="400" type="number">
  Bad request - invalid token or email
</ResponseField>

<ResponseField name="404" type="number">
  Not found - user or token not found
</ResponseField>

<ResponseField name="409" type="number">
  Conflict - email already verified
</ResponseField>

<ResponseField name="500" type="number">
  Internal server error
</ResponseField>

## Notes

- Verification tokens are typically valid for 24 hours
- Users can only verify their email once
- After successful verification, users can access protected endpoints
- If the token expires, users can request a new verification email

## Implementation Example

<CodeGroup>

```javascript JavaScript
// Handle email verification from URL parameters
function handleEmailVerification() {
  const urlParams = new URLSearchParams(window.location.search);
  const token = urlParams.get('token');
  const email = urlParams.get('email');
  
  if (token && email) {
    verifyEmail(token, email);
  }
}

async function verifyEmail(token, email) {
  try {
    const response = await fetch('https://api.railsfromthecrypt.com/v1/auth/verify-email', {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json'
      },
      body: JSON.stringify({
        token: token,
        email: email
      })
    });
    
    const data = await response.json();
    
    if (data.success) {
      // Show success message
      showSuccessMessage('Email verified successfully! You can now log in.');
      // Redirect to login page
      window.location.href = '/login';
    } else {
      // Show error message
      showErrorMessage(data.message);
    }
  } catch (error) {
    showErrorMessage('An error occurred during email verification.');
  }
}
```

</CodeGroup>
