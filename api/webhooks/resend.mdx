---
title: "Resend Webhook"
api: "POST /v1/webhooks/{id}/resend"
description: "Resend a failed webhook delivery"
---

## Overview

Resend a failed webhook delivery. This is useful when a webhook delivery failed due to temporary issues.

## Path Parameters

<ParamField path="id" type="string" required>
  Unique identifier for the webhook
</ParamField>

## Request Body

<ParamField body="log_id" type="string" required>
  ID of the failed log entry to resend
</ParamField>

## Response

<ResponseField name="success" type="boolean">
  Whether the resend was successful
</ResponseField>

<ResponseField name="message" type="string">
  Resend result message
</ResponseField>

<ResponseField name="resend_id" type="string">
  ID of the new delivery attempt
</ResponseField>

<ResponseField name="resend_at" type="string">
  Resend timestamp (ISO 8601)
</ResponseField>

## Example Request

<CodeGroup>

```bash cURL
curl -X POST "https://api.railsfromthecrypt.com/v1/webhooks/wh_123e4567-e89b-12d3-a456-426614174000/resend" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "log_id": "log_123e4567-e89b-12d3-a456-426614174000"
  }'
```

```javascript JavaScript
const response = await fetch('https://api.railsfromthecrypt.com/v1/webhooks/wh_123e4567-e89b-12d3-a456-426614174000/resend', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_ACCESS_TOKEN',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    log_id: 'log_123e4567-e89b-12d3-a456-426614174000'
  })
});

const result = await response.json();
```

</CodeGroup>
