---
title: "Get API Key"
api: "GET /v1/api-keys/{id}"
description: "Get specific API key details by ID"
---

## Overview

Retrieve detailed information about a specific API key using its unique ID.

<Note>
The API key secret is never returned in the response for security reasons.
</Note>

## Path Parameters

<ParamField path="id" type="string" required>
  Unique identifier for the API key
</ParamField>

## Response

<ResponseField name="id" type="string">
  Unique identifier for the API key
</ResponseField>

<ResponseField name="label" type="string">
  API key label
</ResponseField>

<ResponseField name="status" type="string">
  Key status (`active`, `inactive`, `revoked`)
</ResponseField>

<ResponseField name="permissions" type="array">
  Granted permissions
</ResponseField>

<ResponseField name="last_used" type="string">
  Last usage timestamp (ISO 8601)
</ResponseField>

<ResponseField name="created_at" type="string">
  Creation timestamp (ISO 8601)
</ResponseField>

<ResponseField name="expires_at" type="string">
  Expiration timestamp (ISO 8601)
</ResponseField>

<ResponseField name="description" type="string">
  API key description
</ResponseField>

## Example Request

<CodeGroup>

```bash cURL
curl -X GET "https://api.railsfromthecrypt.com/v1/api-keys/ak_123e4567-e89b-12d3-a456-426614174000" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

```javascript JavaScript
const response = await fetch('https://api.railsfromthecrypt.com/v1/api-keys/ak_123e4567-e89b-12d3-a456-426614174000', {
  headers: {
    'Authorization': 'Bearer YOUR_ACCESS_TOKEN'
  }
});

const apiKey = await response.json();
```

</CodeGroup>
