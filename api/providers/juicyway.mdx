## Overview

Receive and process webhook events from Juicyway payment provider. This endpoint handles payment notifications, transaction updates, and other events from Juicyway.

<Note>
This endpoint is designed to receive webhooks from Juicyway and does not require authentication. It's used internally by the system to process provider events.
</Note>

## Request Headers

<ParamField header="X-Juicyway-Signature" type="string" required>
  Webhook signature for verification
</ParamField>

<ParamField header="Content-Type" type="string" required>
  Must be `application/json`
</ParamField>

## Request Body

<ParamField body="event_type" type="string" required>
  Type of event from Juicyway
</ParamField>

<ParamField body="data" type="object" required>
  Event data payload
</ParamField>

<ParamField body="timestamp" type="string" required>
  Event timestamp from Juicyway
</ParamField>

<ParamField body="signature" type="string" required>
  Event signature for verification
</ParamField>

## Response

<ResponseField name="success" type="boolean">
  Indicates if the webhook was processed successfully
</ResponseField>

<ResponseField name="message" type="string">
  Processing result message
</ResponseField>

<ResponseField name="processed_at" type="string">
  Processing timestamp (ISO 8601)
</ResponseField>

## Example Request

<CodeGroup>

```bash cURL
curl -X POST "https://api.railsfromthecrypt.com/v1/providers/juicyway" \
  -H "X-Juicyway-Signature: sha256=abc123..." \
  -H "Content-Type: application/json" \
  -d '{
    "event_type": "payment.completed",
    "data": {
      "transaction_id": "juicyway_txn_123",
      "amount": "500.00",
      "currency": "EUR",
      "status": "completed"
    },
    "timestamp": "2024-01-01T12:00:00Z",
    "signature": "sha256=def456..."
  }'
```

</CodeGroup>

## Example Response

<ResponseExample>

```json Success Response
{
  "success": true,
  "message": "Webhook processed successfully",
  "processed_at": "2024-01-01T12:00:01Z"
}
```

</ResponseExample>
