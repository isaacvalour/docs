## Overview

Retrieve a paginated list of all transactions for the authenticated user. You can filter transactions by status and control pagination with limit and page parameters.

<Note>
Transactions are returned in reverse chronological order (newest first). Use the `status` parameter to filter for specific transaction states.
</Note>

## Query Parameters

<ParamField query="status" type="string">
  Filter transactions by status. Use `pending` for all non-completed transactions or `completed` for completed transactions only.
  
  **Options:**
  - `pending` - All non-completed transactions
  - `completed` - Only completed transactions
  - *(omit for all transactions)*
</ParamField>

<ParamField query="limit" type="number">
  The number of items per page (default: 10, max: 100)
</ParamField>

<ParamField query="page" type="number">
  The page number (default: 1)
</ParamField>

## Example Request

<CodeGroup>

```bash cURL
# Get all transactions
curl -X GET "https://api.railsfromthecrypt.com/v1/transactions" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"

# Get pending transactions only
curl -X GET "https://api.railsfromthecrypt.com/v1/transactions?status=pending" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"

# Get completed transactions with pagination
curl -X GET "https://api.railsfromthecrypt.com/v1/transactions?status=completed&limit=20&page=2" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

async function getRecentTransactions(limit = 10) {

### Get Pending Transactions

<CodeGroup>

```javascript JavaScript
async function getPendingTransactions() {
  try {
    const response = await fetch('https://api.railsfromthecrypt.com/v1/transactions?status=pending', {
      headers: {
        'Authorization': `Bearer ${accessToken}`
      }
    });

    if (!response.ok) {
      throw new Error('Failed to fetch pending transactions');
    }

    const transactions = await response.json();
    
    // Process pending transactions
    transactions.forEach(transaction => {
      console.log(`Pending transaction: ${transaction.reference} - ${transaction.amount} ${transaction.sender_currency}`);
    });
    
    return transactions;
  } catch (error) {
    console.error('Error fetching pending transactions:', error);
    throw error;
  }
}
```

</CodeGroup>

### Paginated Transaction History

<CodeGroup>

```javascript JavaScript
async function getAllTransactions() {
  const allTransactions = [];
  let page = 1;
  let hasMore = true;
  
  try {
    while (hasMore) {
      const response = await fetch(`https://api.railsfromthecrypt.com/v1/transactions?limit=100&page=${page}`, {
        headers: {
          'Authorization': `Bearer ${accessToken}`
        }
      });

      if (!response.ok) {
        throw new Error('Failed to fetch transactions');
      }

      const transactions = await response.json();
      
      if (transactions.length === 0) {
        hasMore = false;
      } else {
        allTransactions.push(...transactions);
        page++;
      }
    }
    
    return allTransactions;
  } catch (error) {
    console.error('Error fetching all transactions:', error);
    throw error;
  }
}
```

</CodeGroup>

### Filter by Date Range

<CodeGroup>

```javascript JavaScript
async function getTransactionsByDateRange(startDate, endDate) {
  try {
    // Get all transactions (you might want to implement date filtering on the server)
    const response = await fetch('https://api.railsfromthecrypt.com/v1/transactions?limit=1000', {
      headers: {
        'Authorization': `Bearer ${accessToken}`
      }
    });

    if (!response.ok) {
      throw new Error('Failed to fetch transactions');
    }

    const transactions = await response.json();
    
    // Filter by date range on client side
    const filteredTransactions = transactions.filter(transaction => {
      const transactionDate = new Date(transaction.created_at);
      return transactionDate >= startDate && transactionDate <= endDate;
    });
    
    return filteredTransactions;
  } catch (error) {
    console.error('Error fetching transactions by date range:', error);
    throw error;
  }
}
```

## Transaction Statuses

<AccordionGroup>
  <Accordion title="Pending">
    Transaction has been created but payment has not been confirmed yet. The user needs to complete the payment to proceed.
  </Accordion>
  <Accordion title="Processing">
    Payment has been confirmed and the transaction is being processed. This includes currency conversion and blockchain operations.
  </Accordion>
  <Accordion title="Completed">
    Transaction has been successfully completed. Funds have been sent to the recipient and the transaction is final.
  </Accordion>
  <Accordion title="Failed">
    Transaction failed due to an error. Check the `reason` field for details about why it failed.
  </Accordion>
  <Accordion title="Cancelled">
    Transaction was cancelled by the user or system before completion.
  </Accordion>
</AccordionGroup>

## Event Tracking

Each transaction includes an events array that tracks the transaction lifecycle:

<Steps>
  <Step title="Awaiting Payment">
    Transaction created, waiting for payment confirmation
  </Step>
  <Step title="Payment Completed">
    Payment has been received and confirmed
  </Step>
  <Step title="Processing">
    Transaction is being processed (conversion, blockchain operations)
  </Step>
  <Step title="Completed">
    Transaction completed successfully
  </Step>
</Steps>

<Info>
Events are automatically updated as the transaction progresses. You can use these events to show real-time status updates to users.
</Info>
