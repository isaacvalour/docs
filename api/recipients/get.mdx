---
title: "Get Recipient"
api: "GET /v1/recipients/{id}"
description: "Get specific recipient details by ID"
---

## Overview

Retrieve detailed information about a specific recipient using their unique ID. This endpoint works for both fiat and cryptocurrency recipients.

<Note>
Sensitive information like account numbers and wallet addresses may be masked for security purposes.
</Note>

## Path Parameters

<ParamField path="id" type="string" required>
  Unique identifier for the recipient
</ParamField>

## Response

<ResponseField name="id" type="string">
  Unique identifier for the recipient
</ResponseField>

<ResponseField name="name" type="string">
  Full name of the recipient
</ResponseField>

<ResponseField name="email" type="string">
  Email address of the recipient
</ResponseField>

<ResponseField name="phone_number" type="string">
  Phone number (fiat recipients only)
</ResponseField>

<ResponseField name="type" type="string">
  Recipient type (`fiat`, `crypto`)
</ResponseField>

<ResponseField name="currency" type="string">
  Currency code
</ResponseField>

<ResponseField name="country" type="string">
  Country code (fiat recipients only)
</ResponseField>

<ResponseField name="bank_name" type="string">
  Bank name (fiat recipients only)
</ResponseField>

<ResponseField name="account_number" type="string">
  Bank account number (masked, fiat recipients only)
</ResponseField>

<ResponseField name="wallet_address" type="string">
  Cryptocurrency wallet address (crypto recipients only)
</ResponseField>

<ResponseField name="network" type="string">
  Blockchain network (crypto recipients only)
</ResponseField>

<ResponseField name="status" type="string">
  Recipient status (`active`, `pending`, `suspended`, `rejected`)
</ResponseField>

<ResponseField name="verification_status" type="string">
  Verification status (`pending`, `verified`, `failed`)
</ResponseField>

<ResponseField name="created_at" type="string">
  Creation timestamp (ISO 8601)
</ResponseField>

<ResponseField name="updated_at" type="string">
  Last update timestamp (ISO 8601)
</ResponseField>

<ResponseField name="metadata" type="object">
  Additional metadata for the recipient
</ResponseField>

## Example Request

<CodeGroup>

```bash cURL
curl -X GET "https://api.railsfromthecrypt.com/v1/recipients/rec_123e4567-e89b-12d3-a456-426614174000" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

```javascript JavaScript
const response = await fetch('https://api.railsfromthecrypt.com/v1/recipients/rec_123e4567-e89b-12d3-a456-426614174000', {
  headers: {
    'Authorization': 'Bearer YOUR_ACCESS_TOKEN'
  }
});

const recipient = await response.json();
```

</CodeGroup>
