---
title: "Delete Recipient"
api: "DELETE /v1/recipients/{id}"
description: "Delete a recipient by ID"
---

## Overview

Delete a recipient from your account. This action cannot be undone and will remove the recipient from all future transfer options.

<Warning>
Deleting a recipient is permanent and cannot be reversed. Ensure you want to remove this recipient before proceeding.
</Warning>

## Path Parameters

<ParamField path="id" type="string" required>
  Unique identifier for the recipient to delete
</ParamField>

## Response

<ResponseField name="success" type="boolean">
  Indicates if the deletion was successful
</ResponseField>

<ResponseField name="message" type="string">
  Confirmation message
</ResponseField>

<ResponseField name="deleted_at" type="string">
  Deletion timestamp (ISO 8601)
</ResponseField>

## Example Request

<CodeGroup>

```bash cURL
curl -X DELETE "https://api.railsfromthecrypt.com/v1/recipients/rec_123e4567-e89b-12d3-a456-426614174000" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

```javascript JavaScript
const response = await fetch('https://api.railsfromthecrypt.com/v1/recipients/rec_123e4567-e89b-12d3-a456-426614174000', {
  method: 'DELETE',
  headers: {
    'Authorization': 'Bearer YOUR_ACCESS_TOKEN'
  }
});

const result = await response.json();
```

</CodeGroup>

## Example Response

<ResponseExample>

## Error Responses

<CardGroup cols={2}>
  <Card title="404 Not Found" icon="search">
    - **RECIPIENT_NOT_FOUND**: Recipient with specified ID not found
  </Card>
  <Card title="401 Unauthorized" icon="shield-exclamation">
    - **INVALID_TOKEN**: Invalid or expired access token
  </Card>
  <Card title="403 Forbidden" icon="lock">
    - **CANNOT_DELETE**: Recipient cannot be deleted (e.g., has pending transfers)
  </Card>
</CardGroup>
